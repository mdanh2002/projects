length (PC units) vs length (bytes) 

No extra space is needed to store data in data memory (e.g. 2 bytes of flash is used for 2 bytes of data)
However, to store constant data in program memory, an extra byte is required due to the need to WORD-align data
and for dsPIC instruction word is 24-bit (3 bytes).
So 3 bytes of flash is required for 2 bytes of data.

Length (PC units) - raw data length
Length (bytes) - flash data length

To avoid wasting the extra byte, one can write codes to access the flash directly.

I measured pin to pin resistance on a bare 44 pin, 64K module and found that AVdd is connected internally to Vdd.
The resistance from AVdd to Vdd is about 2 ohms, the same as between the various Vdd pins.
So it makes no sense to use 2 separate regulators, all the current would be supplied by the highest supply and both would be noisy.
 
There is no continuity between AVss and Vss so AVss can act as a quiet reference if tied to a quiet ground point.
It is interesting to note that the 28 pin module has no AVdd or AVss pins.

https://www.microchip.com/forums/m768005.aspx

(not true for dsPIC33EP512MC502 DIP-28, resistance between AVDD and VDD is around 40K. AVSS and VSS is connected.
AVDD and VDD must come from same power supply.
