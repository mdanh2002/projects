SI4735 crystal critical for accurate frequency tuning
If crystal frequency is wrong, auto tuning might not find stations or find fewer stations even though these stations have strong signals
Manual tuning might also not reach the exact frequency
more obviously with SSB when testing with function generate in sine wave mode. For example, with nominal SSB frequency at 6MHz, 1kHz tone
should be heard in LSB (lower side band) mode if function generate is set to 5.9kHz. If func gen set to 6.1kHz, tone should be heard in USB (upper side band) mode.
Any inaccuracies in crystal freq will result in wrong USB/LSB behaviour
Use a frequency counter or oscilloscope and make sure crystal is at exactly 32768 Hz (some ebay crystals are not)
Voltage threshold might be too low to trigger on some counters/multimeters. Check with oscilloscope.
Auto tuning very sensitive to antenna series capacitor and associated disturbance

We can use T1CK to SI4735 (no need for crystal),
but since T1CK is always on, SI4735 must always be on (otherwise it might be damaged if voltage is passed to it while off). 
So we need to connect SI4735 to 3V3PIC to save power in standby
but I2C must be connected to 3V3SD due to pull-up
Not recommended due to separate power supply

SI4735 pad must use heat gun + solder paste for soldering
Crystal load capacitor not critical
If 32768 crystal not oscillating, I2C interface will work (e.g. version returns 55.48 correctly)
but tuning will fail (cannot read back proper frequency)

Mix solder paste + flux and use a resistor leg to stir
Use the resistor leg to apply solder paste just enough (should not totally cover pads)
Place chip on pad and use heat gun (320C-500C), straight at chip, lowest temperature
Use fine tip soldering iron to fix bridged pins (not too much, else PCB trace will lift)
